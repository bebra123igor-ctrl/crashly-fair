Provably Fair

Открытый код проверки

Это безопасный публичный набор для ручной проверки уже завершённых игр. Тут только deterministic verifier: хеширование, HMAC, расчёт crash, upgrade и dice по уже раскрытым `serverSeed / clientSeed / nonce`.

ОТКРЫТЬ JS-VERIFIER СКАЧАТЬ ФАЙЛ

Что можно публиковать

  • Функции расчёта для уже завершённых раундов.
  • Проверку `serverSeedHash` против раскрытого `serverSeed`.
  • Формулы `dice`, `crash`, `upgrade` и RTP-adjust для проверки результата.
  • Примеры вызова verifier для ручной валидации истории игры.

Что нельзя публиковать

  • Текущие живые `serverSeed` для будущих раундов.
  • `.env`, токены бота, ключи облака, ключи шифрования, дампы БД.
  • Серверный код, который оперирует текущим состоянием игр в реальном времени.
  • Любые приватные сиды до их ротации и раскрытия.

Почему это безопасно

Публичный verifier не даёт угадать будущие результаты, потому что он не содержит текущие активные сиды. Он только повторяет уже известный расчёт для конкретной завершённой игры после раскрытия нужных параметров. Пользователь получает возможность проверить честность, но не возможность влиять на будущий раунд.

import verifier from './provably-fair-verifier.js'; const result = await verifier.verifyCrash({ serverSeed: 'revealed-server-seed', clientSeed: 'client-seed-from-history', nonce: 42, rtpPercent: 100 }); console.log(result.crashPoint, result.serverSeedHash);
